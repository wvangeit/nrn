648 void ShapePlot::draw(Canvas* c, const Allocation& a) const {
649  if (spi_->fast_) {
650 #if defined(WIN32)
651  // win32 clipping is much more strict than X11 clipping even though the
652  // implementations seem to agree that clipping is the intersection of
653  // all clip requests on the clip stack in canvas. Clipping is originally
654  // set to the damage area and thus nothing actually gets drawn to
655  // the buffer. Furthermore, when copying from buffer to screen, only
656  // the original damage area is copied. For some reason in X11 neither
657  // seems to prevent the correct appearance although I wonder if it
658  // will work on all x11 implementations. However, for WIN32, both cause
659  // problems and thus the implementation of Canvas16::clip was changed to
660  // not do an intersection and the "damage_all" below along with a change
661  // to the window.cpp paint function in which the damage area is reread before
662  // the bit block transfer solves the problem.
663  // At any rate this is fast only for the first view since the fast_draw's
664  // only work once.
665 
666  XYView* v = XYView::current_draw_view();
667  c->push_clipping(true);
668 #if defined(WIN32)
669  // Consider the commit message:
670  // -------
671  // 28) Branch: MAIN Date: 2004-05-21 06:30 Commit by: hines
672  // canvas16.cpp
673  // MSWin PolyGlyphs inside a ScrollBox occasionally do not repair all the damage
674  // This is apparently a long standing problem dating from 1999 which just
675  // now became very noticable with the MultipleRunFitter Parameter panel
676  // when there are many parameters (so that it is a scrollbox). This has
677  // been traced to a #if 0 fragment in the clipping mechanism which turned
678  // off the intersection calculation when a new clipping region was requested
679  // We have turned this section back on. There was no reason mentioned in
680  // the 1999 log message when the #if 0 was introduced. The major reason for
681  // the larger change in that file was drawing dashed brush lines. If a
682  // problem with clipping exists then it will have to be fixed in another way.
683  // -------
684  // Well, here is the reason for the #if 0. Without it, the fast_flush movie
685  // is not drawn for shapeplots. (Strangely, the neurondemo movie works for
686  // the stylized neuron but not the pyramidal cell.
687  // So as not to ruin the MAIN 28 commit of canvas16.cpp
688  // we force the clipping to the entire canvas for the fast_flush.
689  // Feeble attempts to do everything here, foundered on my inability to
690  // include <windows.h> in this file. So the implementation is placed in
