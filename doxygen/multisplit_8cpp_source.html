95 // Modification to handle short backbones
96 // Numerical accuracy/stability suffers when backbones are electrically
97 // short. Unfortunately, it is impossible to divide many 3-d reconstructed
98 // cells with any kind of load balance without using them. Also,
99 // unfortunately, it is probably not possible to solve the problem
100 // in general. However we can extend, without introducing
101 // too much parallel inefficiency, the implementation to allow an
102 // arbitrarily short backbone treated as a single node, as long as it
103 // does not connect to a short backbone on another machine. e.g we can
104 // have alternating short and long sausages but not a sequence of
105 // contiguous short sausages. The idea is to exchange backbone info
106 // in two stages. First receive the info that short backbones need
107 // (by definition only from long backbones), compute the exact D,RHS
108 // backbone end points (only a 2x2 matrix solve), and then send the
109 // exact short backbone info. matrix_exchange carries out the
110 // augmented communication. But we need to distinguish short from
111 // long backbones since for short ones we need to factor out the
112 // first part of bksub_backbone and do it in the middle of
113 // matrix_exchange (as well as avoid doing that twice in bksub_backbone).
114 // We can accomplish that by introducing the index
115 // backbone_long_begin between the backbone_begin and backbone_interior_begin
116 // indices. Then in matrix exchange we do backbone_begin to
117 // backbone_long_begin - 1 , and in bksub_backbone, the first part is
118 // from backbone_long_begin to backbone_interior_begin - 1.

120 // Modification to handle any number of backbones, short or long, exactly.
121 // In this case, it is not necessary to consider numerical stability/accuracy.
122 // However a tree matrix with rank equal to the number of distinct sid for
123 // an entire parallelized cell must be solved between the send and receive
124 // stages for that cell. The main issue to be resolved is: which machine receives
125 // the matrix information. ie. a single sid subtree sends d and rhs to the

2559  // D=sum d and RHS=sum rhs
2560  // the info for zero area if nonzero should be D=1.0 and RHS=v
2561  // and the info for nonzero should be D=sum d and RHS=sum rhs.
