1716 static PyObject* iternext_sl(PyHocObject* po, hoc_Item* ql) {
1717  // Note that the longstanding behavior of changing the currently
1718  // accessed section during iteration no longer takes place because
1719  // we cannot guarantee that an iterate will complete with state
1720  // PyHoc::Last and so the previous Section would have been left on the
1721  // hoc section stack.
1722 
1723  // Primarily the Section is pushed and the currently accessed python
1724  // Section is returned during sequential iteration over the list ql.
1725  // On re-entry here, the previous Section is popped.
1726  // The complexity is due to the possibility that the returned nrn_Section
1727  // may be deleted with h.delete_section(sec=nrn_Section). This would
1728  // invalidate the po->iteritem_ (point to freed memory) if it were
1729  // the iteritem of the current Section. Thus we choose to store
1730  // the next iteritem pointer in the list. Although not 100% safe, since
1731  // the user body of the iterator is allowed to delete_section an arbitary
1732  // subset of sections, the obvious work around, making a copy of ql,
1733  // is considered not worth it.
1734 
1735  // In this implementation, the first call to internext_sl starts out
1736  // in state PyHoc::Begin with po->iteritem_ == ql. If there is no valid
1737  // item, it sets po->iteritem_ = NULL and returns NULL. If there is
1738  // a valid item, it moves to state PyHoc::Last or PyHoc::NextNotLast
1739  // depending on whether there is a valid secitem following the current
1740  // item. The current section is pushed and currently accessed python
1741  // section is returned.
1742 
1743  // Thereafter, re-entry with po->iteritem_ == NULL, immediately
1744  // returns NULL.
1745 
1746  // Re-entry in state PyHoc::NextNotLast, pops the previously pushed Section,
1747  // sets sec to the current Section from po->iteritem_, and
1748  // sets po->iteritem_ to the next_valid_section. If there is no next_valid
1749  // section, then move to state PyHoc::Last.
1750  // Push the current section and return currently accessed python Section.
1751 
1752  // Re-entry in state PyHoc::Last just pops the current section, sets
