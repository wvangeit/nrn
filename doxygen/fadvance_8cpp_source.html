19 /*
20  after an fadvance from t-dt to t, v is defined at t
21  states that depend on v are defined at t+dt/2
22  states, such as concentration that depend on current are defined at t
23  and currents are defined at t-dt/2.
24  */
25 /*
26  fcurrent is used to set up all assigned variables without changing any
27  states. It assumes all states have their values at time t which is
28  only first order correct. It determines the nonvint assignments first
29  so that ena, etc. are correct for the current determinations. We
30  demand that nonvint assignments can be done without changing states when
31  dt is temporarily set to 0.
32 
33  It turned out to be a bad idea to do this. Many methods (KINETIC, PROCEDURE)
34  for solving are not even called when dt =0. Also it plays havoc with tables
35  that are recomputed several times when dt is changed then changed back.
36  Therefore we are no longer trying to initialize assigned variables within
37  SOLVE'd blocks. Instead we are making initialization more convenient,
38  at least within the normal situations, by introducing a finitialize()
39  with an optional argument vinit. finitialize will call the INITIALIZE block
40  for all mechanisms in all segments. (Default initialization sets all states
41  to state0.) The user can set things up specially in a models INITIAL
42  block. INITIAL blocks can make use of v. With care they can make use of
43  ionic concentrations just like breakpoint and solve blocks. When the argument
44  is present, v for all segments are set to that value.
45  */
